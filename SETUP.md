# Project Vidya — Developer Setup Guide

Welcome to the development team! This guide will walk you through setting up the Project Vidya modular monolith platform on your local laptop.

---

## 🛠️ Prerequisites

Before you begin, ensure you have the following installed on your machine:
1. **Node.js**: version `22.x` or higher.
2. **pnpm**: version `11.x` or higher (pnpm is pinned via packageManager).
3. **Docker**: for running Postgres, Redis, and MinIO storage containers.

---

## 🚀 Setup Steps

### 0. Get the Code (Clone)
Once you've been added as a collaborator on GitHub, clone the repository and enter it:
```bash
git clone https://github.com/<owner>/<repo>.git
cd <repo>
```
*(Replace `<owner>/<repo>` with the actual repository path the owner shares with you.)*

### 1. Install Dependencies
At the root of the project, run:
```bash
pnpm install --frozen-lockfile
```

### 2. Configure Environment Variables
Copy the template `.env.example` file to `.env`:
```bash
# On Linux/macOS:
cp .env.example .env

# On Windows (PowerShell):
Copy-Item .env.example .env
```
*Note: The default environment variables are already pre-configured to point to the local Docker services.*

### 3. Spin Up Dependency Services (Docker)
Ensure Docker/Docker Desktop is running on your machine, then spin up the database, cache, and object storage:
```bash
pnpm compose:up
```
*Note for developers: This starts the core containers (`postgres`, `redis`, `minio`). If the Next/Worker build inside Docker fails or takes too long, don't worry—the database & cache containers will still start and run successfully in the background, allowing you to run Web & Worker processes locally on your host machine.*

Verify they are running:
```bash
docker compose ps
```

### 4. Run Schema Migrations
Prepare the database schema by running migrations. If you are on Windows, ensure environment variables from `.env` are loaded:
```powershell
# On Windows (PowerShell) to load env variables:
node --env-file=.env node_modules/tsx/dist/cli.mjs scripts/migrate.ts status
node --env-file=.env node_modules/tsx/dist/cli.mjs scripts/migrate.ts up

# On Linux/macOS:
pnpm db:status
pnpm db:migrate
```

### 5. Seed the Demo Dataset
To populate your local database with realistic departments, classes, subjects, teachers, students, mock attendances, and marks, run the demo seeder:
```powershell
# On Windows (PowerShell):
$env:VIDYA_ALLOW_DEMO_SEED="true"
node --env-file=.env node_modules/tsx/dist/cli.mjs scripts/seed-demo.ts

# On Linux/macOS:
VIDYA_ALLOW_DEMO_SEED=true pnpm seed:demo
```
Once completed, it will print a list of credentials corresponding to different roles (admin, principal, HOD, teacher, class teacher, accountant, student).

### 6. Run the Developer Servers
You need to run the Next.js web application and the background worker process simultaneously.

#### A) Start the Web App Server
```powershell
# On Windows (PowerShell) loading env variables:
Get-Content .env | Foreach-Object { if ($_ -match "^([^#\s=]+)=(.*)$") { [System.Environment]::SetEnvironmentVariable($Matches[1], $Matches[2]) } }
pnpm dev

# On Linux/macOS:
pnpm dev
```
The web dashboard will be available at: **`http://localhost:3000`**

#### B) Start the Background Worker Process
In a separate terminal tab:
```powershell
# On Windows (PowerShell):
node --env-file=.env node_modules/tsx/dist/cli.mjs watch apps/worker/src/main.ts

# On Linux/macOS:
pnpm dev:worker
```
The worker observability details will be available at: **`http://localhost:9464/ready`**

---

## 🧪 Verification & Testing

Verify that both systems are running healthy:
- Web ready check: `curl http://localhost:3000/ready`
- Web health check: `curl http://localhost:3000/health`
- Worker ready check: `curl http://localhost:9464/ready`

To run the unit/frontend testing suites locally:
```bash
pnpm test       # Unit tests
pnpm test:ui    # Frontend React Testing Library tests on JSDOM
```
